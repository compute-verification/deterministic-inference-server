# Experiment log: spec-decode

Started 2026-06-05. See demos/spec-decode/plan.md.

- 2026-06-05: Phase 0 (local, mock, no GPU). Built the speculative-decoding demo as a third task-graph scenario on the proof-compare PR. `spec_decode.py` implements real greedy speculative decoding over two pluggable `next_token` functions (`mock_models` for CPU, `hf_models` for GPU) plus shared `run_mock`/`run_hf` helpers so host and recomp do byte-identical work. Key correctness lever: greedy spec-decode is output-identical to plain greedy target decoding, asserted in tests via `greedy_decode`. New graph builder `build_spec_decode_task_graph()` (in modules/proof_server/task_graph.py) produces the new shape — a committed spine (accepted drafts + corrections, linked by `next`) with rejected drafts as pruned dead-ends, and two FLOP weight-classes (draft 2·N_draft vs verify 2·N_target·(K+1)); added Qwen3-0.6B to MODEL_DIMS as the draft. Full 5-process topology mirroring the inference demo, with its own `servers/` copy (envelope carries the per-round trace; recomp /verify returns its recomp_payload; tap forwards both payloads to the proof server /compare, which compares output ids + trace and builds the graph). Tests: tests/unit/test_spec_decode.py (5: output==greedy, speedup, accept pattern) + 7 spec-graph cases in test_task_graph.py. `demo.sh --mock` → ALL PASS (2 compared, 2 matches, 2 graphs built; e.g. 5 target passes for 16 tokens). Full unit suite 547 passed / 2 skipped. Phase 1 (real GPU run on vast.ai + viz with real data) next.
- 2026-06-05: Phase 1 (real GPU, vast.ai H100). Launched a fresh H100 (~$2/hr, instance 39571380) with the `hiyouga/llamafactory:latest` image (torch 2.6 + transformers 5.6 preinstalled) rather than the Nix project image — much less setup for a torch+transformers run. Reached the box via the **direct** SSH route (public_ipaddr:50039); the vast SSH **proxy** (sshN.vast.ai) refused connections (AWS proxy closed the socket), a recurring vast quirk — use the direct host:port from `show instance --raw` `ports['22/tcp']`. Shipped a 19KB minimal bundle (demos/spec-decode + modules/proof_server/task_graph + modules/core/common/deterministic) with a **neutralized empty modules/__init__.py** so importing `modules.proof_server.task_graph` doesn't trigger the real `modules/__init__.py` (which imports the heavy Pipeline). Pre-downloaded Qwen3-0.6B + Qwen3-1.7B to warm the HF cache. Two roadblocks fixed in demo.sh: (1) starting host+recomp concurrently aborted the CUDA context (4 model loads at once) → serialize startup (host warms, then recomp); (2) fixed `sleep 1.0` before the health check was too short for real-model recomp (re-runs the whole decode, ~seconds) so the assertion fired at compared=0 and `exit 4` skipped the graph copy → replaced with a poll loop on the proof server until compared>=2. Real run `demo.sh --real` → ALL PASS: id=1 MATCH, real Qwen3 output " jumps over the lazy dog. 100000000000" (30-node graph: 6 rounds, 24 drafts = 13 accepted / 11 rejected, 6 target passes for 16 tokens ≈ 1.83× tokens/pass); host vs recomp output ids + per-round trace bitwise-identical on the same box. Pulled the real graph back and added a third "Speculative decoding" tab to the viz (committed spine + pruned rejected stubs + fat target verify nodes), republished to https://taskgraph-dss.surge.sh. Instance destroyed.
